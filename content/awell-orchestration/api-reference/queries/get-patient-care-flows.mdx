---
title: Get patient care flows
description: Get all care flows a given patient is or was enrolled in
---

## Query

```graphql
query GetPatientPathways($patient_id: String!, $status: [String!]) {
  patientPathways(
    patient_id: $patient_id
    filters: { status: { in: $status } }
  ) {
    patientPathways {
      id
      title
      pathway_definition_id
      status
      status_explanation
      version
      release_id
      baseline_info {
        value
        definition {
          id
          title
          category
          key
          valueType
          possibleValues {
            label
            value
          }
          unit
          range {
            min
            max
          }
        }
      }
    }
  }
}
```

## Variables

### Required parameters

You can identify the patient using either `patient_id` or `patient_identifier`.

#### Using patient_id

```json
{
  "patient_id": "{{ PATIENT_ID }}"
}
```

#### Using patient_identifier

```json
{
  "patient_identifier": {
    "system": "{{ IDENTIFIER_SYSTEM }}",
    "value": "{{ IDENTIFIER_VALUE }}"
  }
}
```

### Filters

The filter is optional: if you don't provide one, all care flows for the patient will be returned.

```graphql
query GetPatientPathways($patient_id: String!) {
  patientPathways(patient_id: $patient_id) {
    patientPathways {
      ...Pathway
    }
  }
}
```

The `in` filter type for status accepts a list of values and results in an `is any of` search criteria.

```json
{
  "patient_id": "{{ PATIENT_ID }}",
  "status": ["active", "completed"]
}
```

#### Status

You can use the following status values in the `status` filter array.

```graphql
enum PathwayStatus {
  active
  completed
  missing_baseline_info
  starting
  stopped
}
```

### Pagination and sorting

There is no pagination or sorting available for this query. All matching patient care flows will be returned.
